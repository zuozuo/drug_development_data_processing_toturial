## 1. 先搞懂：什么是"域"？

### 用文件夹类比

想象你要整理一个人的全部医疗档案。如果把所有东西堆在一起，会非常乱：

```
❌ 乱七八糟的档案：
体检报告、身份证复印件、验血单、吃过的药清单、
过敏反应记录、既往病史、血压记录、心率记录...
全部混在一起，根本找不到！
```

聪明的做法是**按类别分文件夹**：

```
✅ 整理后的档案（每个文件夹 = 一个"域"）：

📁 基本信息/（DM 域）
   └── 身份证复印件、年龄、性别、民族...

📁 生命体征/（VS 域）
   └── 血压记录、心率记录、体温记录...

📁 化验检查/（LB 域）
   └── 血常规、尿常规、肝肾功能...

📁 不良反应/（AE 域）
   └── 头痛记录、恶心记录、皮疹记录...

📁 用药记录/（EX 域）
   └── 研究药物的用药记录...

📁 其他用药/（CM 域）
   └── 同时在吃的其他药物...

📁 既往病史/（MH 域）
   └── 以前得过什么病...
```

**"域"（Domain）就是 SDTM 中的"文件夹"，把同一类数据放在一起。**

### 一句话定义

> **域 = 一张标准化的数据表，存储某一类临床数据**

每个域都有：
- **固定的名称**（2个字母，如 DM、VS、AE）
- **固定的结构**（该有哪些列/变量）
- **固定的规则**（每个变量怎么填）

## 2. SDTM 的域分类：三大家族

SDTM 的所有域分为三大"家族"，每个家族的结构不太一样：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      SDTM 域的三大家族                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 1️⃣ 特殊用途域（Special Purpose）                            │   │
│  │                                                             │   │
│  │    只有 DM（人口学）属于这一类                                │   │
│  │    特点：每个患者只有一行，存储基本信息                        │   │
│  │                                                             │   │
│  │    📋 DM = Demographics（人口学）                            │   │
│  │       → 患者的"身份证"：年龄、性别、种族等                    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 2️⃣ 事件域（Events）                                         │   │
│  │                                                             │   │
│  │    记录"发生了什么事"，每件事一行                             │   │
│  │    特点：有开始日期、结束日期                                 │   │
│  │                                                             │   │
│  │    ⚠️ AE = Adverse Events（不良事件）                        │   │
│  │       → 吃药后出现的不良反应                                  │   │
│  │                                                             │   │
│  │    📜 MH = Medical History（病史）                           │   │
│  │       → 以前得过什么病                                       │   │
│  │                                                             │   │
│  │    💊 CE = Clinical Events（临床事件）                        │   │
│  │       → 其他临床事件                                         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 3️⃣ 发现域（Findings）                                       │   │
│  │                                                             │   │
│  │    记录"检查/测量结果"，每次检查一行                          │   │
│  │    特点：有检测项目、检测结果、单位                           │   │
│  │                                                             │   │
│  │    🩺 VS = Vital Signs（生命体征）                           │   │
│  │       → 血压、心率、体温                                     │   │
│  │                                                             │   │
│  │    🧪 LB = Laboratory Tests（实验室检查）                    │   │
│  │       → 血常规、肝功能、肾功能                                │   │
│  │                                                             │   │
│  │    ❤️ EG = ECG（心电图）                                     │   │
│  │       → 心电图检查结果                                       │   │
│  │                                                             │   │
│  │    👁️ PE = Physical Examination（体格检查）                  │   │
│  │       → 身体检查结果                                         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │ 4️⃣ 干预域（Interventions）                                  │   │
│  │                                                             │   │
│  │    记录"给患者做了什么"                                      │   │
│  │    特点：有剂量、频率、持续时间                               │   │
│  │                                                             │   │
│  │    💊 EX = Exposure（用药暴露）                              │   │
│  │       → 研究药物的使用记录                                   │   │
│  │                                                             │   │
│  │    💊 CM = Concomitant Medications（合并用药）               │   │
│  │       → 同时使用的其他药物                                   │   │
│  │                                                             │   │
│  │    🔧 PR = Procedures（操作）                                │   │
│  │       → 手术、治疗操作                                       │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 简单记忆

| 家族 | 回答什么问题 | 典型域 |
|------|-------------|-------|
| **特殊用途** | 这个患者是谁？ | DM |
| **事件** | 发生了什么事？ | AE、MH |
| **发现** | 检查结果是什么？ | VS、LB、EG |
| **干预** | 给患者做了什么？ | EX、CM、PR |

### 这些变量几乎每个域都有

不管是哪个域，有一些变量是"通用"的，几乎每个域都会用到：

```

│  ─────────────────────────────────                                  │
│                                                                     │

│                                                                     │

│              例：XXXBP001-001                                        │

│              只在本研究内唯一                                        │

## 4. 核心域详解（一）：DM 人口学域

### 这个域是什么？

**DM = Demographics = 人口学**

存储患者的基本信息，就像患者的"身份证"。

### 核心特点

```
DM 域的特点
─────────────

1. 每个患者只有 1 行（其他域可能有多行）
2. 是所有域的"基础"，其他域都要参考它
3. 包含患者的基本信息和试验关键日期
```

### 关键变量详解

| 变量名 | 英文全称 | 中文含义 | 示例 | 说明 |
|--------|----------|----------|------|------|
| **STUDYID** | Study Identifier | 研究编号 | XXXBP001 | 哪个临床试验 |
| **DOMAIN** | Domain | 域名 | DM | 固定是 DM |
| **USUBJID** | Unique Subject ID | 唯一受试者标识 | XXXBP001-001 | 患者的"全球身份证" |
| **SUBJID** | Subject ID | 受试者编号 | 001 | 研究内编号 |
| **SITEID** | Site ID | 中心编号 | 01 | 哪个医院 |
| **AGE** | Age | 年龄 | 58 | 数值 |
| **AGEU** | Age Unit | 年龄单位 | YEARS | 通常是 YEARS |
| **SEX** | Sex | 性别 | M | M=男, F=女 |
| **RACE** | Race | 种族 | ASIAN | 亚洲人 |
| **ETHNIC** | Ethnicity | 民族 | NOT HISPANIC | 非西班牙裔 |
| **COUNTRY** | Country | 国家 | CHN | 中国 |
| **RFSTDTC** | Reference Start Date | 参考开始日期 | 2024-01-15 | 通常是首次给药日期 |
| **RFENDTC** | Reference End Date | 参考结束日期 | 2024-04-15 | 通常是最后给药日期 |
| **ARM** | Planned Arm | 计划治疗组 | Drug A 10mg | 分到哪个治疗组 |
| **ARMCD** | Arm Code | 治疗组代码 | DRUGA10 | 治疗组的代码 |

```

• 003 号患者：55 岁男性，02 中心，分到 Drug A 10mg 组，1月17日开始用药
```

### 小白常见问题

**Q: 为什么 SEX 用 M/F 而不是"男/女"？**

因为 SDTM 是国际标准，用英文代码可以全球通用。M = Male（男），F = Female（女）。

**Q: RFSTDTC 为什么叫"参考开始日期"？**

它是计算其他日期的"参考点"。比如，"第 4 周访视"是相对于这个日期算的。

## 5. 核心域详解（二）：VS 生命体征域

### 这个域是什么？

**VS = Vital Signs = 生命体征**

存储患者的生命体征检查结果：血压、心率、体温、呼吸等。

### 核心特点

```
VS 域的特点
─────────────

1. 每次检查一行（同一患者可能有很多行）
2. 属于"发现域"（Findings），有固定的结果-单位结构
3. 是降压药、心血管药物试验中的核心数据
```

### 关键变量详解

| 变量名 | 英文全称 | 中文含义 | 示例 | 说明 |
|--------|----------|----------|------|------|
| **VSTESTCD** | VS Test Code | 检查项目代码 | SYSBP | 收缩压的代码 |
| **VSTEST** | VS Test Name | 检查项目名称 | Systolic Blood Pressure | 收缩压的全称 |
| **VSORRES** | VS Original Result | 原始结果 | 152 | 实际测量值 |
| **VSORRESU** | VS Original Unit | 原始结果单位 | mmHg | 毫米汞柱 |
| **VSSTRESC** | VS Standard Result (Char) | 标准结果（字符） | 152 | 标准化后的结果 |
| **VSSTRESN** | VS Standard Result (Num) | 标准结果（数值） | 152 | 用于计算的数值 |
| **VSSTRESU** | VS Standard Unit | 标准单位 | mmHg | 标准化后的单位 |
| **VSSTAT** | VS Status | 检查状态 | | 空=完成, NOT DONE=未做 |
| **VSPOS** | VS Position | 体位 | SITTING | 坐位测量 |
| **VSLOC** | VS Location | 测量位置 | ARM | 手臂 |
| **VSDTC** | VS Date/Time | 检查日期时间 | 2024-01-15 | ISO 格式 |
| **VISITNUM** | Visit Number | 访视编号 | 1 | 第几次访视 |
| **VISIT** | Visit Name | 访视名称 | SCREENING | 筛选期 |

### 常见检查项目代码

```
VS 域常用 VSTESTCD
──────────────────

血压相关：
• SYSBP  = Systolic Blood Pressure（收缩压，"高压"）
• DIABP  = Diastolic Blood Pressure（舒张压，"低压"）

心率相关：
• HR     = Heart Rate（心率）
• PULSE  = Pulse Rate（脉率）

其他：
• TEMP   = Temperature（体温）
• RESP   = Respiratory Rate（呼吸频率）
• HEIGHT = Height（身高）
• WEIGHT = Weight（体重）
• BMI    = Body Mass Index（体质指数）
```

### 结果变量的区别

```
VSORRES vs VSSTRESN 的区别
───────────────────────────

VSORRES（原始结果）：
• 存储医生/仪器记录的原始值
• 可能是字符型（如 "152" 或 ">400"）
• 保留原始信息

VSSTRESN（标准数值结果）：
• 转换成标准单位后的数值
• 只能是数字
• 用于统计分析

为什么需要两个？
假设 A 医院用 "mmHg"，B 医院用 "kPa"（千帕）
• VSORRES 保留原始：152 mmHg 或 20.3 kPa
• VSSTRESN 统一换算成 mmHg：都是 152
```

## 6. 核心域详解（三）：LB 实验室检查域

### 这个域是什么？

**LB = Laboratory Tests = 实验室检查**

存储血液、尿液等化验检查的结果。

### 核心特点

```
LB 域的特点
─────────────

1. 数据量通常最大（一次抽血可能测几十项指标）
2. 属于"发现域"，结构和 VS 域类似
3. 需要特别注意正常值范围
```

### 关键变量详解

| 变量名 | 英文全称 | 中文含义 | 示例 | 说明 |
|--------|----------|----------|------|------|
| **LBTESTCD** | LB Test Code | 检查项目代码 | ALT | 谷丙转氨酶代码 |
| **LBTEST** | LB Test Name | 检查项目名称 | Alanine Aminotransferase | 谷丙转氨酶全称 |

### 正常范围的重要性

```
为什么 LB 域特别关注正常范围？

## 7. 核心域详解（四）：AE 不良事件域

### 这个域是什么？

**AE = Adverse Events = 不良事件**

存储患者在试验期间出现的任何不良反应或健康问题。

### 核心特点

```
AE 域的特点
─────────────

1. 属于"事件域"，有开始和结束日期
2. 是安全性评估的核心数据
3. 需要判断与研究药物的关系
4. 可能影响药物是否能获批
```

### 关键变量详解

| 变量名 | 英文全称 | 中文含义 | 示例 | 说明 |
|--------|----------|----------|------|------|
| **AETERM** | AE Reported Term | 报告的不良事件名称 | 头痛 | 医生记录的原始描述 |
| **AEDECOD** | AE Dictionary Coded | 标准编码名称 | Headache | 用标准词典编码后的名称 |
| **AEBODSYS** | AE Body System | 系统器官分类 | Nervous system disorders | 神经系统疾病 |
| **AESEV** | AE Severity | 严重程度 | MILD | 轻度/中度/重度 |
| **AESER** | AE Serious | 是否严重 | N | Y=严重不良事件，N=非严重 |
| **AEREL** | AE Relationship | 与药物的关系 | POSSIBLY RELATED | 可能相关 |
| **AEACN** | AE Action Taken | 采取的措施 | DOSE NOT CHANGED | 剂量未变 |
| **AEOUT** | AE Outcome | 转归 | RECOVERED | 已恢复 |
| **AESTDTC** | AE Start Date | 开始日期 | 2024-01-20 | 什么时候出现 |
| **AEENDTC** | AE End Date | 结束日期 | 2024-01-25 | 什么时候好转/结束 |
| **AESTDY** | AE Start Study Day | 开始的研究日 | 6 | 相对于首次给药第几天 |
| **AEENDY** | AE End Study Day | 结束的研究日 | 11 | 相对于首次给药第几天 |

### 严重程度分级

```
AESEV（严重程度）取值
─────────────────────

MILD（轻度）：
• 症状轻微
• 不影响日常活动
• 例：轻微头痛，休息一下就好

MODERATE（中度）：
• 症状明显
• 影响日常活动
• 例：头痛需要吃止痛药

SEVERE（重度）：
• 症状严重
• 无法进行日常活动
• 例：剧烈头痛，卧床不起

注意：SEVERE ≠ SERIOUS
• SEVERE 是严重程度（多严重）
• SERIOUS 是是否为严重不良事件（是否危及生命/住院等）
```

### 与药物关系的判断

```
AEREL（与药物的关系）取值
──────────────────────────

NOT RELATED（无关）：
• 确定与研究药物无关
• 例：车祸骨折

UNLIKELY RELATED（可能无关）：
• 可能性很小
• 例：药物不太可能引起这种反应

POSSIBLY RELATED（可能相关）：
• 有一定可能性
• 时间上有关联，但不确定

PROBABLY RELATED（很可能相关）：
• 可能性较大
• 时间关联紧密，符合药物特点

DEFINITELY RELATED（肯定相关）：
• 确定是药物引起的
• 有明确证据
```

```

• 001 患者：
  - 第 6 天出现头痛，轻度，可能与药物相关，2天后恢复
  - 第 7 天出现恶心，轻度，很可能与药物相关，2天后恢复

• 002 患者：

```

### 严重不良事件（SAE）

```
什么是严重不良事件（SAE）？
──────────────────────────

AESER = Y 的情况（任何一条满足即为 SAE）：
1. 导致死亡
2. 危及生命
3. 需要住院或延长住院
4. 导致永久或显著的残疾
5. 先天畸形/出生缺陷
6. 其他重要医学事件

SAE 需要立即报告给药监局！
即使临床试验还没做完，发现 SAE 也必须马上报告。
```

## 8. 核心域详解（五）：EX 用药暴露域

### 这个域是什么？

**EX = Exposure = 用药暴露**

存储患者使用研究药物（正在测试的药物）的记录。

### 核心特点

```
EX 域的特点
─────────────

1. 只记录研究药物，不记录其他药物（其他药物在 CM 域）
2. 属于"干预域"，有剂量、频率、持续时间

4. 与疗效分析密切相关

## 9. 核心域详解（六）：CM 合并用药域

### CM 域的重要作用

```
为什么 CM 域很重要？
────────────────────

   可能影响疗效或产生不良反应

2. 解释异常结果

### 为什么需要记录病史？

```
病史的重要作用
──────────────

   • 了解患者的健康状况
   • 帮助解释后续的治疗反应

   • 比较"有糖尿病史"和"无糖尿病史"患者的疗效差异

4. 安全性评估
   • 某些病史可能增加不良反应风险
   • 例：有肝病史的患者可能更容易出现肝损伤
```

## 附录：常见问题解答

### Q1: SDTM 和 ADaM 有什么区别？

```
SDTM vs ADaM
────────────

SDTM = 标准化存储
• 把原始数据整理成标准格式
• 变量名、代码统一
• 适合长期保存和审计

ADaM = 分析就绪
• 在 SDTM 基础上增加派生变量
• 如：基线值、变化值、分析标志
• 可以直接用于统计分析

关系：CRF → SDTM → ADaM
```

### Q2: 为什么要用这么多英文缩写？

```
使用英文缩写的原因
──────────────────

1. 国际标准：全球通用，不依赖特定语言
2. 变量名限制：SAS 变量名最多 8 个字符
3. 一致性：所有公司、所有国家都用同样的名称
4. 高效：审评员看到 SYSBP 就知道是收缩压
```

### Q3: 我只想了解核心内容，哪些必须掌握？

```
最核心的内容（必须掌握）
──────────────────────

1. DM 域：理解 USUBJID 的作用
2. VS 域：理解 ORRES/STRESN 的区别
3. AE 域：理解 AESEV 和 AESER 的区别
4. 变量命名规则：能看懂新变量的含义
```

### Q4: 实际工作中最常出错的地方？

```
常见错误
────────

1. 日期格式错误：应该用 ISO 8601 格式（YYYY-MM-DD）
2. 代码不规范：SEX 应该是 M/F，不是 1/2 或 男/女
3. USUBJID 格式不一致：应该全局唯一
4. 单位换算错误：ORRES 和 STRESN 单位不匹配
5. 缺失值处理不当：应该留空，不是填 "N/A"
```
