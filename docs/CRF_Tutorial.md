## 1. 一个故事：为什么需要 CRF？

### 想象你是一个医生

假设你要同时管理 100 个高血压病人的治疗记录：

```
❌ 没有统一表格的情况：

病人张三的记录：
  "今天血压有点高，给他开了降压药"

病人李四的记录：
  "BP: 150/95，用药：XXX 10mg"

病人王五的记录：
  "患者自述头晕，测了一下血压，偏高"

问题：
• 每个医生记录方式不同
• 有的写了具体数值，有的没写
• 有的记录了用药剂量，有的没记
• 无法统计、无法比较、无法分析！
```

```
✅ 有统一表格的情况：

┌──────────────────────────────────────────────────────┐
│                    血压记录表                         │
├────────┬──────────┬──────────┬──────────┬───────────┤
│ 病人ID │   日期   │ 收缩压   │ 舒张压   │  用药情况  │
├────────┼──────────┼──────────┼──────────┼───────────┤
│ 001    │ 2024-1-1 │ 155      │ 95       │ XXX 10mg  │
│ 002    │ 2024-1-1 │ 150      │ 92       │ XXX 10mg  │
│ 003    │ 2024-1-1 │ 148      │ 90       │ XXX 5mg   │
└────────┴──────────┴──────────┴──────────┴───────────┘

优点：
• 格式统一，每个医生填的内容一样
• 数据完整，必填项不能漏
• 可以统计、比较、分析！
```

### CRF 就是临床试验的"统一记录表"

当一个新药要在 20 家医院、1000 个病人身上做临床试验时，必须有一套统一的表格，确保所有数据采集标准一致。

**这套统一的表格，就叫 CRF。**

## 2. CRF 是什么？一句话解释

**CRF = Case Report Form = 病例报告表**

> **一句话定义**：临床试验中用来记录每个受试者（病人）所有相关数据的标准化表格。

### 类比理解

| 场景 | 类比 |
|------|------|
| 体检 | CRF 就像"体检表"，记录身高、体重、血压、血液指标等 |
| 考试 | CRF 就像"答题卡"，所有考生用同样的格式填写 |
| 调查问卷 | CRF 就像"标准问卷"，确保每个人回答同样的问题 |
| 病历 | CRF 是"标准化病历"，比普通病历更规范、更完整 |

### CRF 回答的核心问题

```
CRF 要记录的信息：
─────────────────────

1. 这个病人是谁？
   → 人口学信息（年龄、性别、身高、体重等）

2. 病人符合入组条件吗？
   → 入选/排除标准核查

3. 病人吃了什么药？
   → 研究用药记录、合并用药记录

4. 病人的身体指标怎么变化？
   → 生命体征、实验室检查、疗效评估

5. 病人有没有不舒服？
   → 不良事件记录

6. 病人完成试验了吗？
   → 试验完成情况、退出原因
```

```
  │ 试验方案 │─────────▶│病例报告表│─────────▶│ 标准化  │────────▶│ 药监局  │
  └─────────┘          └─────────┘          └─────────┘         └─────────┘
       │                    │                    │

       ▼                    ▼                    ▼
  "规定采集什么"      "实际采集数据"        "转换成标准格式"
```

### CRF 是数据的"源头"

```
数据溯源链：
─────────────────────────────────────────────────────────

提交给药监局的数据
        ↑
        │ 来自
        │
    ADaM 分析数据
        ↑
        │ 派生自
        │
    SDTM 标准化数据
        ↑
        │ 转换自
        │
  ┌─────────────────┐
  │      CRF        │  ← 数据的源头！
  │  病例报告表      │
  │  (原始采集数据)   │
  └─────────────────┘
        ↑
        │ 填写
        │
    研究者/医生
```

**重要概念**：CRF 是"源数据"（Source Data），所有后续的数据处理都可以追溯到 CRF。

### 对比

| 特性 | 纸质 CRF | 电子 CRF (eCRF/EDC) |
|------|---------|---------------------|
| **填写方式** | 手写 | 电脑/平板录入 |
| **数据传输** | 快递/传真 | 实时上传 |
| **数据验证** | 人工核查 | 系统自动验证 |
| **错误发现** | 事后发现 | 即时提示 |
| **数据查询** | 电话/邮件 | 系统内质疑 |
| **成本** | 印刷、快递、存储 | 系统费用 |
| **现状** | 逐渐淘汰 | **主流方式** |

## 8. CRF 设计的基本原则

### 8.1 数据最小化原则

```
只采集必要的数据
─────────────────────

❌ 错误：采集一切可能有用的数据
   → 数据太多，增加工作量
   → 可能采集到无关隐私数据

✅ 正确：只采集 Protocol 规定的、分析需要的数据
   → 数据精简
   → 减少研究者负担
```

### 8.2 标准化原则

```
使用标准化的格式和术语
─────────────────────────────

❌ 不好的设计：
   "请描述不良事件：_______________"
   （自由文本，每个人写法不同）

✅ 好的设计：
   "不良事件名称：_______________"
   "严重程度：□ 轻度  □ 中度  □ 重度"
   "与药物关系：□ 有关  □ 可能有关  □ 无关"
   （结构化采集，便于统计）
```

### 8.3 可溯源原则

```
每条数据都能追溯到源头
─────────────────────────────

CRF 数据 ←── 必须能对应到 ──► 源文档（病历、检验报告）

例如：
• CRF 上血压 = 150/95 mmHg
• 源文档（检查记录）上也应该有 150/95 mmHg 的记录
```

### 8.4 完整性原则

```
数据要完整，不能有空缺
─────────────────────────────

所有字段都要有值：
• 有数据 → 填写数据
• 无数据 → 填写原因（如"未做"、"不适用"）
• 不能留空白！
```

## 11. 常见问题

### Q1: CRF 和病历有什么区别？

| 方面 | 病历 | CRF |
|------|------|-----|
| **用途** | 临床诊疗记录 | 临床试验数据采集 |
| **内容** | 所有诊疗信息 | 只记录试验相关信息 |
| **格式** | 各医院不同 | 全国/全球统一 |
| **标准化** | 相对自由 | 高度标准化 |
| **监管** | 医疗机构管理 | 药监局审查 |

### Q2: 为什么 CRF 要设计得这么详细？

```
原因：确保数据质量
─────────────────────

1. 标准化
   → 100 家医院、1000 个病人，数据格式一致
   → 可以汇总分析

2. 完整性
   → 不会漏掉重要信息
   → 必填项强制填写

3. 可追溯
   → 每条数据都能追溯到源头
   → 药监局可以核查

4. 合规性
   → 符合 GCP（药物临床试验质量管理规范）
   → 符合监管要求
```

## 12. 总结

### 一图看懂 CRF

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│                         CRF 核心要点                                │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   是什么？    病例报告表 —— 临床试验的"标准化数据采集表"              │
│                                                                     │
│   为什么？    确保数据标准化、完整、可追溯、可分析                     │
│                                                                     │
│   谁来填？    研究者（医生/护士）填写，CRA 核查，数据管理录入          │
│                                                                     │
│   填什么？    ┌─────────────────────────────────────┐              │
│              │  • 受试者基本信息（人口学）            │              │
│              │  • 入选/排除标准                      │              │
│              │  • 研究用药记录                       │              │
│              │  • 疗效评估（生命体征、实验室等）      │              │
│              │  • 安全性评估（不良事件）              │              │
│              │  • 试验完成情况                       │              │
│              └─────────────────────────────────────┘              │
│                                                                     │
│   现在主流？  电子 CRF（eCRF），通过 EDC 系统在线填写                 │
│                                                                     │
│   后续流程？  CRF → SDTM（标准化）→ ADaM（分析）→ 提交药监局          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 关键要点

1. **CRF 是数据的源头** —— 所有分析数据都可以追溯到 CRF
2. **标准化是核心** —— 统一格式确保数据可比较、可汇总
3. **eCRF 是主流** —— 现代临床试验几乎都用电子系统
4. **CRF 与 Protocol 紧密相关** —— Protocol 规定采集什么，CRF 执行采集
5. **CRF 数据要转换成 SDTM** —— 转换后才能提交药监局

### 一句话记住 CRF

> **CRF 就是临床试验的"标准化体检表"，确保全国各地的医院用同样的方式记录同样的信息，这样才能汇总分析、得出可靠结论。**
